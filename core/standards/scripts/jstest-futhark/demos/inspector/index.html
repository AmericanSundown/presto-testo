Author

Server side support

The DOM inspector works by injecting some (quite a lot, in fact) script code into a page. For security reasons, this is not something that can actually be done straight away on the client side. There are probably two main ways to solve this:

  • Adding special support for it in Opera.
  • Letting a simple proxy service insert a script tag into a page and serve the resulting page to the client.

The second alternativ is the one I have used so far. There is a Python script (inspector.py) that asks for an url, loads the url, inserts a script tag (and a base tag to make relative urls work as intended) and sends the result to the client.

The Python script itself is a very quick hack, and will most likely not work with all pages (or work at all in all cases). But it works in some cases, and that is good enough, right now.

The script is currently available via my home computer on my student Internet connection. It probably won't be there forever, especially since I'm already not actually a student.

Known problems

The inspector highlights the currently selected element by adding a one-pixel border and changing its color to different colours (all red). Properly restoring the border when the element is no longer selected requires DOM CSS support that Opera 7 currently doesn't have. This results in borders being left or old borders not being restored.

It generally doesn't seem to work very well. That is something we'll have to fix.
