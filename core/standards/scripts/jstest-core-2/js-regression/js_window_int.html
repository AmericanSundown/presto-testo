The popup window is for reports from the different event handlers.

User interaction

Event handlers

Try to trigger the following event handlers:
  1. onBlur(), remove focus from the window, clicking on a different window on your desktop.
  2. onDragDrop(), when the user drops an object onto the browser window, such as dropping a file.
  3. onError(), when the loading of a document or image causes an error. Trigg an error
  4. onFocus(), should be triggered when loading the page, and when the window receives focus after been blurred ( another window hsa had the focus ).
  5. onLoad(), should be triggered when loading the page, and when reloading it.
  6. onMove(), should be triggered when when the user or script moves a window or frame. Move this window
  7. onResize(), should be triggered when a user or script resizes a window or frame. Resize this window horizontally Resize this window vertically
  8. onUnload(), should be triggered when closing the window, or entering a different URL. Go back in history

Properties not tested in the non interactive test

  1. pageXOffset. Provides the current x-position, in pixels, of a window's viewed page. Get current pageXOffset
  2. pageYOffset. Provides the current y-position, in pixels, of a window's viewed page. Get current pageYOffset

  6. forward(). Moves one forward in the document history ( if there is one ).
  7. home(). Moves to the page defined as your homepage.
  8. print(). Prints the contents of the window or frame.
  9. prompt(). If you press 'Ok', the text in the input field is returned, if you press 'Cancel', null is returned.

  10. scroll().
    sccrollTo().

    scrollTo() replaced scroll() in JS 1.2, but the scroll() remains for backward compatibility. Thus, both methods should behave identically, that is to: Scrolls the viewing area of the window so that the specified point becomes the top-left corner.

    The test: both scroll() and scrollTo() opens two windows; one 100x200 and one 100x100. The first one should scroll down 200 pixels on the y axis, so that red box is the first thing you see in your top left corner. The second window is a square that is to be scrolled 200 pixels, both on the x and y axis. Thus, you should lower right corner of the cyan square in the top left corner of the window.

  11. scrollBy(). Scrolls the contents of the window a specified amount of the x, y. It differs from scroll() and scrollTo() in that it scrolls relative to where the viewarea is. Scrollbars shoud be visible in order to get any effect of this method.

    The test opens two windows; one 100x200 and one 100x100. The first one starts at x = 0 and y = 200, then uses scrollBy() to go down 300 on the axis, you should see the lower left corner of the second, red box. The second uses scrollBy() to go 200 on the x axis and 200 on the y axis. Thus, you shuld see the lower right corner of the first box in your top left corner of the window.
