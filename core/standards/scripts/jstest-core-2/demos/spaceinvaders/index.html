SpaceInvaders

-SpaceInvaders made with DOM and JavaScript

ASCII version

This version uses PRE elements with ascii inside. CSS in an external stylesheet is used to describe how they are too look. All figures move inside a big table element. Play!

  • Heavy manipulation of a document. The cells in the table are constantly changed, many simultaneously. Evertything on screen moves by altering the contents of this table.

Image version

This version uses transparent PNG images inside DIV elements for figures. These are moved around by setting their top and left CSS properties in pixels. Thus, they can move around all the screen that is given the game. Play!

  • DOM/CSS. The figures are moved through DOM/CSS, changing the top and left CSS properties of the different elements. Also, setAttribute() is used numerous time to change both normal element attributes like id, but also to set/change external CSS information, like "class".
  • Heavy rendering. After a few levels, the game/document gets extremly heavy to render. Very much is going on at the same time, with numerous images (20+) moving constantly.
  • Preloading of images, using the Image object, see the init() method.

Both versions

  • DOM/HTML. Almost everything is done through the DOM. The only thing which are not DOM, is the starting documents, which are pretty clean, and the external stylesheet, residing in styles/spaceinvaders.css
  • Sound. Both version uses the DOM to create elements for playing sound. If the browser used is either Mozilla or Galeon, an EMBED element is created, whereas BGSOUND is created if Opera is being used. bug #83554
  • Mouse events. The ship is controlled with the mouse, shooting is done by clicking. Thus, the application tests onmousemove and onclick mouse events.
  • Keyboard events. just game commands like "pause" and "do you wish to continue y/n"
  • Concurrence. A lot of setTimeout() and setInterval(). Everything that has a predictable motion, has a setInterval(), such as bombs, missiles and alien movement. A lot of setTimeout() is used, e.g. typewriting and explosions.
  • Currently, these demos works best in Mozilla 1.0/1.1 and Galeon 1.2.5. The ASCII version works *pretty good* in Opera 7 build 2131 and IE 6 now plays both versions ( after some modifcations to the code ), but has huge problems whith transparent PNG images, so the figures on screen look a little odd.
  • IE issues I encountered when making the game play in IE 5.5/6 Unsupported JavaScript methods and properties, difference in DOM implementation and how to work around them in Explorer.
  • Browser support for event object properties. This is an overview made when developing the SpaceInvaders. It handles the browsers Opera 6, Opera 7, Mozilla 1.1 and Internet Explorer 6 and their support for the different event object properties.

Code

  • spaceinvaders.js
    This file is used both for the ascii version and the image version of SpaceInvaders. All game logic that is version independent goes in here.
  • conifg.js
    All global variables used by both versions reside in this file.
  • objects.js
    All object definitions goes in this file. These include Missile, AlienBomb, Base and BaseUnit. Used by both versions of SpaceInvaders.
  • lib.js
    Code that is not directly connected to the SpaceInvaders is put in here. E.g. browser sniffing.
  • spaceinvaders-ascii.js
    Game logic specific to the ASCII version of SpaceInvaders goes in here.
  • config-ascii.js
    Global variables used only by the ASCII version reside here.
  • spaceinvaders-image.js Game logic specific to the Image version of SpaceInvaders goes in here.
  • config-image.js Global variables used only by the Image version reside here.
