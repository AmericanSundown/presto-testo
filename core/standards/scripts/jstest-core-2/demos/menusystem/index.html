Demo menu system

This is a rather straight forward menu system, which can be configured to use various DHTML features to create the HTML for the menus and for handling events.

Author

Jens Lindström <jl@opera.com> wrote this. It's quite possible that there are bugs in the menu system. If you find one, please report it.

Versions

There are three versions:

menusystem-document.write.html
Uses document.write to create the HTML and registers event handlers using HTML attributes (<... onclick="...">). Works in Opera 6, but doesn't look so nice.
menusystem-innerHTML.html
Uses innerHTML and insertAdjacentHTML to create the HTML and registers event handlers by assigning to event properties (element.onlick = ...).
menusystem-dom2.html
Uses createElement, createTextNode and appendChild to create the HTML and registers event handlers using addEventListener.

The three versions only differ in assignments to properties on a global object 'features' which controls how the code in menusystem.js behaves. The different possible configurations are documented in the beginning of menusystem.js.

Known problems

The document.write-based example doesn't work (it is probably the registering of event handlers as HTML attributes that doesn't work). The problem causes an alert to be displayed with an error message in.

If you are using MSIE for some reason, you need to add "features.MSIE_compatible = true;" somewhere, as MSIE's event objects are a bit limited. That should work with Opera too, and possibly with Mozilla.
